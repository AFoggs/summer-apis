Learn Data APIs and Full-Stack JavaScript

Spend the early part of your summer interacting with web-available data APIs for exchanging and mashing up content and data, while extending your skills in JavaScript to the server with Node.js.

This lectureless, discussion-based course will open with a study of APIs in the classical sense: as interfaces into programming languages and core modules, specifically JavaScript. An understanding and command of JavaScript’s APIs, both in the browser and server-side in Node.js, will allow us to explore and manipulate APIs that present data in lightweight data serialization formats (e.g., JSON) in a RESTful architecture. Students will have the opportunity to focus on a specific use of APIs, such as visualization, “scraping” for the purposes of creating datasets for research, or building custom and novel visual interfaces for interacting with a particular dataset or service. Throughout the course, students will encounter and learn to address challenges surrounding licensing, API request limits, and other legal, ethical, and technical issues surrounding terms of service (TOS) and documentation.

The Hard Sell: Take This Class

Whether or not you have professional ambitions in web development, there is no escaping the importance of accessing and manipulating data APIs. Additionally, there are numerous other higher-level principles and practices that this class introduces or reinforces:

  • The professional necessity of tracking the development of active open-source projects
  • Better understanding of the processes of authenticating to data services and acting within the bounds of terms of service
  • Engaging in collaborative, open development on GitHub while maintaining the security of assets such as API keys and tokens
  • Exposure to JavaScript-based methods and patterns for asynchronous, non-blocking I/O
  • Full-stack development that closely knits the front- and back-end logic and execution through standards-compliant HTML and client- and server-side JavaScript
  • Intensive work with consoles and loggers to up your debugging game outside of clients and browsers
  • Test-driven development (TDD) to ensure code quality and internal consistency, and to streamline code reviews

Week-by-Week Overview

  1. Week 1: JavaScript and Web Fundamentals

    • HTML, CSS, and JavaScript syntax, structures, and style
    • Fundamental browser-embedded JavaScript objects
    • Browser-parsed data in JavaScript Object Notation (JSON)
    • Navigating and referencing documentation

  2. Week 2: API Structure, Endpoints

    • Data serialization formats
    • Navigating, referencing, and sanity-checking API documentation
    • RESTful HTTP request/response architecture

  3. Week 3: Node.js Fundamentals

    • Fundamental Node.js modules and objects
    • EcmaScript 6 JavaScript syntax
    • Mocking API responses locally for development purposes
    • Asynchronous, non-blocking I/O on local and remote data access
    • Developing test suites for test-driven development (TDD)

  4. Week 4: The Structure of Node.js Applications

    • The Express.js minimalist web-application framework
    • Structuring apps in Express.js
    • Templating options for views
    • Passing data from API requests into views

  5. Week 5: Authenticating to APIs

    • Per-app vs per-user authentication
    • Keeping authentication assets secure
    • API terms of service

  6. Week 6: Application Deployment

    • Automated deployment with pm2
    • Monitoring application status and performance
    • Caching API requests

Book List

  • Haverbeke, Marijn. Eloquent JavaScript. 2nd ed. No Starch Press. $39.95 (eBook). ISBN 9781593275846

    Required. A thorough and excellent book on JavaScript, whether you’re new to the language or in need of a refresher and opinionated guidance.

  • Wilson, Jim R. Node.js 8 the Right Way: Practical, Server-Side JavaScript that Scales. Pragmatic Bookshelf. $19.95 (eBook). ISBN 9781680501957

    Required. A thorough and well written guide to Node.js. Buy the electronic edition from Prag to continue to receive updates (as with all Pragmatic Bookshelf titles).

  • Swicegood, Travis. Pragmatic Version Control Using Git. Pragmatic Bookshelf. $25.00 (eBook). ISBN 9781934356159

    Not required, unless you have little to no experience with command-line Git, or want a refresher. Recommended read ahead of class.

My policy for assigning books: they are all required in the edition indicated; the total retail pricetag for the entire course should be less than $100 (this one is $84.90 on the high end, assuming you opt for the electronic editions at full list price and the optional title); and each book should be worthy of a place on your bookshelf or electronic device of choice long after the class has ended.

Major Projects

There will be three major projects in this class: An individually developed browser-based app, an individually developed server-side alpha app, and a collaboratively developed server-side beta app. Additionally, graduate students will be expected to more deeply investigate both a core module from Node.js and a common data serialization format (e.g., JSON, XML, CSV, SOAP, YAML) and serve as in-house consultants on them.

Required Tech Setup

  • A Unix-like operating system (Linux, BSD, macOS, etc.). Virtualized Linux is fine, but Windows is not.
  • A plain-text editor that highlights HTML, CSS, JavaScript syntax. Atom is a fine choice, if you have no preferred editor or are looking to try something new.
  • The Node.js JavaScript runtime.
  • A command-line Git installation and GitHub account.

Full setup details will be emailed to registered students ahead of the first week of class.
